# Prozess-Steckbrief Technical Architecture Management

**Bezug zu IT-Strategie /Auftrag**

- Sicherstellung einer stabilen, skalierbaren technischen Infrastruktur (Digital MoveS)
- Harmonisierung der historisch gewachsenen Systemlandschaft
- Grundlage für sichere, zukunftsfähige IT-Betriebsplattformen

**Prozesszweck** Planung, Pflege und Steuerung der technischen IT-Infrastruktur (Hardware, Netzwerke, Plattformen) zur Sicherstellung von Stabilität, Skalierbarkeit und Sicherheit.

**Prozessziele, KPI, Werte**

- Aktualität der technischen Architekturdokumentation
- Standardisierungsgrad der Infrastrukturkomponenten
- Verfügbarkeit der technischen Plattformen
- Reduktion technischer Schulden / Altsysteme

**Prozess Inputs durch Lieferanten**

- Architekturvorgaben → Enterprise Architecture Management
- Kapazitätsanforderungen → Capacity Management
- Sicherheitsanforderungen → IT Security Management
- Anforderungen / Projekte → IT-Projektmanagement
- Markttrends → IT Research Management

**Prozessaktivitäten**

- Technische Standards definieren
- Infrastrukturarchitektur planen
- Konformität prüfen
- Projekte technisch beraten
- Architektur fortschreiben

**Prozess Outputs an Kunde**

- Technisches Architekturkonzept → IT-Projektmanagement
- Infrastrukturstandards → IT-Betrieb
- Konformitätsbewertung → Change Management
- Zielarchitektur → Enterprise Architecture Mgt
- Beratungsergebnis → Fachbereiche

**Kritische Erfolgsfaktoren**

- Verbindlichkeit der Standards
- Frühzeitige Projekteinbindung
- Aktuelle Technologiebeobachtung
- Abstimmung mit Security & EAM

**Prozess Beteiligte / Rollen**

- Technical Architect → Steuerung
- IT-Betrieb → Umsetzung
- Enterprise Architect → Abstimmung
- IT Security Manager → Vorgaben

**Unterstützende IT-Systeme**

- EA-Tool (z.B. LeanIX)
- CMDB
- Dokumentenmanagement
