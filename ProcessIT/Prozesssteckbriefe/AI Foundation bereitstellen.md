# Prozess-Steckbrief: AI Foundation bereitstellen

**Bezug zu IT-Strategie / Auftrag**

- Stuttgart als Vorreiterin im KI-Einsatz gemäß KI-Strategie
- Stadtweiter, skalierbarer Ausbau von KI-Anwendungen
- Grundlage für sichere und innovative Verwaltungsleistungen

**Prozesszweck** Bereitstellung von Infrastruktur, Plattformen und Tools als stabile, skalierbare Basis für Entwicklung und Betrieb von KI-Lösungen.

**Prozessziele, KPI, Werte**

- Verfügbarkeit der KI-Infrastruktur
- Anzahl unterstützter KI-Anwendungsfälle
- Skalierbarkeit der Plattform
- Compliance-Konformität (Governance)

**Prozess Inputs durch Lieferanten**

- KI-Strategie → Strategy Management
- Anforderungen KI-Anwendungsfälle → Fachbereiche
- Architekturvorgaben → IT-Architekturmanagement
- Sicherheitsvorgaben → IT Security Management
- Kapazitätsdaten → Capacity Management

**Prozessaktivitäten**

- KI-Infrastruktur bereitstellen
- Plattformen und Tools konfigurieren
- Zugriffsrechte und Governance verwalten
- Skalierung und Kapazität überwachen
- Fachbereiche beraten und unterstützen

**Prozess Outputs an Kunde**

- Betriebsbereite KI-Plattform → Application Development
- KI-Tools/Modelle → Fachbereiche
- Governance-Vorgaben → IT-Leitung
- Betriebsreport → IT-Management
- Beratungsergebnis → KI-Hub

**Prozess Beteiligte / Rollen**

- AI Platform Manager
- Solution Architect
- IT Security Manager
- KI-Hub Vertreter

**Unterstützende IT-Systeme**

- KI-Plattform (z.B. Azure AI, GitHub Copilot)
- IAM-/Access-Tool
- Monitoring-Tool

**Kritische Erfolgsfaktoren**

- Skalierbare, sichere Infrastruktur
- Enge Anbindung an KI-Hub
- Klare Governance-Regeln
- Kontinuierliche Kapazitätsplanung
