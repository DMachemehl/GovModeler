# Prozess-Steckbrief: Application Development & Software Engineering

**Bezug zu IT-Strategie /Auftrag**

- Bereitstellung qualitativ hochwertiger, wartbarer Software gemäß Digitalisierungsstrategie
- Effiziente Steuerung des gesamten Softwarelebenszyklus
- Grundlage für Innovation und Modernisierung der Anwendungslandschaft

---

**Prozesszweck** Systematische Planung, Entwicklung, Testung und Wartung von Softwarelösungen zur Erfüllung von Nutzer- und Geschäftsanforderungen.

---

**Prozessziele, KPI, Werte**

- Fehlerquote in Produktion
- Time-to-Market neuer Funktionen
- Testabdeckungsgrad
- Einhaltung Entwicklungsstandards

---

**Prozess Inputs durch Lieferanten**

- Servicedesign-Dokument → IT Service Design
- Anforderungen → IT Anforderungs Mgt
- Architekturvorgaben → Anwendungsarchitektur
- Entwicklungsumgebung → Entwicklungsumgebung bereitstellen
- Freigegebener Change → Change Management

---

**Prozessaktivitäten**

- Anforderungen analysieren
- Software entwickeln
- Software testen
- Software dokumentieren
- Wartung & Weiterentwicklung durchführen

---

**Prozess Outputs an Kunde**

- Lauffähige Software → Release Management
- Technische Dokumentation → Configuration Management
- Testergebnisse → Change Management
- Wartungsbericht → IT-Betrieb

---

**Kritische Erfolgsfaktoren**

- Klare Anforderungsdefinition
- Einhaltung von Standards & Methoden
- Ausreichende Testabdeckung

---

**Prozess Beteiligte / Rollen**

- Software Engineer
- Product Owner
- QA/Tester
- Solution Architect

---

**Unterstützende IT-Systeme**

- IDE/CI-CD-Plattform
- Versionsverwaltung (Git)
- Testmanagement-Tool
