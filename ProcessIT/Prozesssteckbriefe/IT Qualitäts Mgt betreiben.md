# Prozess-Steckbrief: IT-Qualitätsmanagement betreiben

**Bezug zu IT-Strategie

- Hohe Qualität & Standards der IT-Leistungen
- Steigerung der Anwenderzufriedenheit
- Kontinuierliche Verbesserung (Digital MoveS)

**Prozesszweck:** Planung, Überwachung und Verbesserung der Qualität von IT-Prozessen, -Dienstleistungen und -Produkten zur Sicherstellung hoher Standards und Minimierung von Fehlern.

**Prozessziele, KPI, Werte:**

- Anwenderzufriedenheit
- Fehler-/Reklamationsquote
- Prozesskonformitätsgrad
- Umsetzungsquote Verbesserungen

**Prozess Inputs durch Lieferanten:** 

- Qualitätsvorgaben → IT-Leitung
- Prozessdaten → Prozessverantwortliche
- Feedback → Anwender/Fachbereiche
- Audit-Ergebnisse → Governance
- KPI-Daten → Performance Mgt

**Prozessaktivitäten:** 

- Qualitätsstandards festlegen
- Qualität messen & prüfen
- Abweichungen analysieren
- Verbesserungsmaßnahmen ableiten
- Wirksamkeit überwachen

**Prozess Outputs an Kunde:** 

- Qualitätsberichte → IT-Leitung
- Verbesserungsmaßnahmen → Prozessverantwortliche
- Qualitätsstandards → IT-Betrieb
- Audit-Vorgaben → Fachbereiche
- Reviews → CSI

**Prozess Beteiligte / Rollen:** 

- Quality Manager → Steuerung Qualität
- Prozessverantwortliche → Umsetzung
- IT-Leitung → Vorgaben/Freigabe
- Anwender → Feedback

**Unterstützende IT-Systeme:** 

- QM-Tool
- QlikSense
- Excel

**Kritische Erfolgsfaktoren:**

- Gelebte Qualitätskultur
- Klare Standards/Kriterien
- Regelmäßige Audits
- Management-Unterstützung
