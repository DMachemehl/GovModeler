# Prozess-Steckbrief: Entwicklungsumgebung bereitstellen

**Bezug zu IT-Strategie / Auftrag**

- Sichere, effiziente Arbeitsumgebung für Softwareentwicklung gemäß Digital MoveS
- Standardisierung von Tools und Plattformen zur Skalierbarkeit
- Grundlage für qualitätsgesicherte Application Development-Prozesse

**Prozesszweck** Einrichtung, Konfiguration und Bereitstellung stabiler, sicherer Entwicklungsumgebungen inkl. Tools zur Unterstützung effizienter Softwareentwicklung.

**Prozessziele, KPI, Werte**

- Bereitstellungszeit neuer Entwicklungsumgebungen
- Verfügbarkeit der Entwicklungsplattformen
- Standardisierungsgrad genutzter Tools
- Entwicklerzufriedenheit mit Arbeitsumgebung

**Prozess Inputs durch Lieferanten**

- Anforderungen Entwicklungsumgebung → Application Development
- Architekturvorgaben → IT-Architekturmanagement
- Sicherheitsvorgaben → IT Security Management
- Lizenzinformationen → IT Procurement
- Kapazitätsdaten → Capacity Management

**Prozessaktivitäten**

- Anforderungen erfassen und analysieren
- Entwicklungsumgebung konfigurieren und einrichten
- Versionierungssystem und Zugriffsrechte verwalten
- Testumgebung bereitstellen
- Nutzungsregelungen dokumentieren und kommunizieren

**Prozess Outputs an Kunde**

- Betriebsbereite Entwicklungsumgebung → Application Development
- Zugriffsrechte / Berechtigungen → Entwickler
- Testumgebung → Software Engineering
- Nutzungsrichtlinien → Entwicklerteams
- Statusreport → IT-Management

**Prozess Beteiligte / Rollen**

- DevOps Engineer
- Solution Architect
- IT Security Manager
- Entwickler / Application Owner

**Unterstützende IT-Systeme**

- Versionierungssystem (z.B. GitLab)
- CI/CD-Plattform
- IAM-/Access-Tool

**Kritische Erfolgsfaktoren**

- Standardisierte Toolauswahl
- Schnelle Bereitstellung
- Klare Zugriffsregelungen
- Enge Abstimmung mit Security
