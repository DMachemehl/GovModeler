# Prozess-Steckbrief: IT Availability Management

**Bezug zu IT-Strategie /Auftrag**

- Sicherstellung stabiler, zuverlässiger IT-Systeme gemäß IT-Strategie
- Gewährleistung vereinbarter Servicequalität für die Verwaltung
- Grundlage für Vertrauen und Verlässlichkeit digitaler Services

**Prozesszweck** Sicherstellung der vereinbarten Verfügbarkeit von IT-Services durch präventive Maßnahmen und schnelle Wiederherstellung.

**Prozessziele, KPI, Werte**

- Servicverfügbarkeit (%) gemäß SLA
- Mean Time Between Failures (MTBF)
- Mean Time To Restore (MTTR)
- Anzahl SLA-Verstöße

**Prozess Inputs durch Lieferanten**

- SLA-Vorgaben → Service Level Mgt
- Incident-Daten → Incident Mgt
- Kapazitätsdaten → Capacity Mgt
- Monitoring-Daten → IT Betrieb
- Risikobewertung → IT Security Mgt

**Prozessaktivitäten**

- Verfügbarkeitsanforderungen ermitteln
- Verfügbarkeit überwachen
- Risiken analysieren
- Präventivmaßnahmen umsetzen
- Ausfälle nachbereiten

**Prozess Outputs an Kunde**

- Verfügbarkeitsberichte → Service Level Mgt
- Risikoanalysen → IT Security Mgt
- Präventivmaßnahmen → IT Betrieb
- SLA-Reports → IT-Management
- Verbesserungsvorschläge → CSI

**Kritische Erfolgsfaktoren**

- Proaktives Monitoring
- Klare Eskalationswege
- Redundante Systemarchitektur
- Enge Abstimmung mit Capacity Mgt

**Prozess Beteiligte / Rollen**

- Availability Manager
- IT Betrieb
- Service Level Manager
- IT Security Manager

**Unterstützende IT-Systeme**

- Monitoring-Tool
- ITSM-Tool
- Power BI
