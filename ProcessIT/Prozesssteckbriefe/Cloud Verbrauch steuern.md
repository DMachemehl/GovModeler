# Prozess-Steckbrief: Cloud Verbrauch steuern

**Bezug zu IT-Strategie /Auftrag**

- Wirtschaftlicher, richtlinienkonformer Einsatz von Cloud-Ressourcen gemäß IT-Strategie
- Sicherstellung von Kostentransparenz und Skalierbarkeit der Cloud-Nutzung
- Unterstützung nachhaltiger, bedarfsgerechter IT-Leistungserbringung

**Prozesszweck** Überwachung, Analyse und Steuerung der Cloud-Ressourcennutzung zur Kostenoptimierung und Richtlinieneinhaltung.

**Prozessziele, KPI, Werte**

- Cloud-Kostenabweichung (Soll-Ist)
- Auslastungsgrad Cloud-Ressourcen
- Compliance-Quote zu Cloud-Richtlinien
- Anzahl umgesetzter Optimierungsmaßnahmen

**Prozess Inputs durch Lieferanten**

- Verbrauchsdaten → Cloud Provider
- Budgetvorgaben → IT Financial Mgt
- Cloud-Richtlinien → IT Governance
- Kapazitätsbedarf → Capacity Mgt
- Vertragskonditionen → Vendor Mgt

**Prozessaktivitäten**

- Verbrauch überwachen
- Kosten analysieren
- Richtlinienkonformität prüfen
- Ressourcen optimieren
- Reporting erstellen

**Prozess Outputs an Kunde**

- Kostenberichte → IT Financial Mgt
- Optimierungsempfehlungen → IT Betrieb
- Compliance-Report → IT Governance
- Verbrauchsprognosen → Capacity Mgt
- Nutzungsreport → Fachbereich

**Kritische Erfolgsfaktoren**

- Transparente Kostenmodelle
- Automatisiertes Monitoring
- Klare Cloud-Richtlinien
- Enge Abstimmung mit Financial Mgt

**Prozess Beteiligte / Rollen**

- Cloud Manager
- IT Controller
- Compliance Officer
- Service Owner

**Unterstützende IT-Systeme**

- Cloud Management Platform
- Cost-Monitoring-Tool (z.B. Azure Cost Mgt)
- Power BI
