# Prozess-Steckbrief: IT Capacity Management

**Bezug zu IT-Strategie /Auftrag**

- Sicherstellung bedarfsgerechter IT-Ressourcen gemäß IT-Strategie
- Wirtschaftlicher Einsatz technischer Kapazitäten
- Grundlage für Skalierbarkeit und Zukunftsfähigkeit der IT

**Prozesszweck** Analyse, Planung und Optimierung technischer IT-Ressourcen zur Erfüllung aktueller und zukünftiger Geschäftsanforderungen.

**Prozessziele, KPI, Werte**

- Auslastungsgrad der IT-Ressourcen
- Planungsgenauigkeit Kapazitätsbedarf
- Anzahl Kapazitätsengpässe
- Kosteneffizienz der Ressourcennutzung

**Prozess Inputs durch Lieferanten**

- Bedarfsprognosen → Demand Mgt
- Auslastungsdaten → IT Betrieb
- Serviceportfolio → Service Portfolio Mgt
- Verfügbarkeitsanforderungen → Availability Mgt
- Wachstumsplanungen → Strategy Mgt

**Prozessaktivitäten**

- Kapazitätsbedarf analysieren
- Auslastung überwachen
- Kapazitätsplan erstellen
- Ressourcen optimieren
- Engpässe frühzeitig identifizieren

**Prozess Outputs an Kunde**

- Kapazitätsplan → IT-Management
- Auslastungsberichte → Availability Mgt
- Optimierungsmaßnahmen → IT Betrieb
- Investitionsempfehlungen → IT Financial Mgt
- Prognosen → Strategy Mgt

**Kritische Erfolgsfaktoren**

- Verlässliche Bedarfsprognosen
- Kontinuierliches Monitoring
- Enge Abstimmung mit Betrieb
- Frühzeitige Engpasserkennung

**Prozess Beteiligte / Rollen**

- Capacity Manager
- IT Betrieb
- Availability Manager
- IT-Financial Mgt

**Unterstützende IT-Systeme**

- Monitoring-Tool
- Power BI
- ITSM-Tool
