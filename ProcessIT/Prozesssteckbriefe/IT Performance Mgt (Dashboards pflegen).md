# Prozess-Steckbrief: IT Performance Management (Dashboards pflegen)

**Bezug zu IT-Strategie / Auftrag:** 

- Datenbasierte Steuerung der IT (Digital MoveS)
- Transparenz über IT-Leistung & Qualität
- Kennzahlensteuerung mittels Dashboards

**Prozesszweck:** _(max. 180)_ Messung, Analyse und Steuerung der Leistungsfähigkeit von IT-Systemen und -Services sowie Bereitstellung der Erkenntnisse über Dashboards zur kontinuierlichen Verbesserung.

**Prozessziele, KPI, Werte:** 

- Service-Verfügbarkeit
- KPI-Abdeckungsgrad
- Dashboard-Aktualität
- Umsetzungsquote Maßnahmen

**Prozess Inputs durch Lieferanten:** 

- Leistungsdaten → IT-Betrieb
- SLA-Werte → Service Level Mgt
- Incident-Daten → Incident Mgt
- KPI-Vorgaben → IT-Leitung
- Kostendaten → IT-Financial Mgt

**Prozessaktivitäten:** 

- KPIs definieren
- Daten erfassen & konsolidieren
- Leistung analysieren
- Dashboards pflegen
- Optimierungsmaßnahmen ableiten

**Prozess Outputs an Kunde:**

- Performance-Dashboards → Management
- KPI-Berichte → IT-Leitung
- Optimierungsmaßnahmen → IT-Betrieb
- Trendanalysen → Strategy Mgt
- Reports → Fachbereiche

**Prozess Beteiligte / Rollen:**

- Performance Manager → Analyse/KPIs
- Data Analyst → Dashboards
- IT-Leitung → Steuerung
- Service Owner → Datenlieferung

**Unterstützende IT-Systeme:**

- QlikSense/Power BI
- Monitoring-Tools
- Excel

**Kritische Erfolgsfaktoren:**

- Verlässliche Datenbasis
- Klar definierte KPIs
- Automatisierte Datenerfassung
- Akzeptanz im Management
