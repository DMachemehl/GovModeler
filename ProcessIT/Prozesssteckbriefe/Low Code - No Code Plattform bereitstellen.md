# Prozess-Steckbrief: Low Code / No Code Plattform bereitstellen

**Bezug zu IT-Strategie / Auftrag**

- Beschleunigung der IT-Lösungsentwicklung gemäß Digital MoveS
- Befähigung der Fachbereiche zur eigenständigen Anwendungserstellung
- Förderung von Innovation und Zukunftsfähigkeit

**Prozesszweck** Einführung und Betrieb von Low Code / No Code Plattformen zur schnellen, eigenständigen Anwendungserstellung durch Fachbereiche.

**Prozessziele, KPI, Werte**

- Anzahl erstellter Fachbereichsanwendungen
- Time-to-Solution für Fachanwendungen
- Nutzerzufriedenheit Fachbereiche
- Governance-Konformität erstellter Lösungen

**Prozess Inputs durch Lieferanten**

- Anforderungen → Fachbereiche
- Architekturvorgaben → IT-Architekturmanagement
- Sicherheitsvorgaben → IT Security Management
- Plattformlizenzen → IT Procurement
- Governance-Regeln → IT-Leitung

**Prozessaktivitäten**

- Plattform auswählen und bereitstellen
- Zugriffsrechte und Rollen verwalten
- Fachbereiche schulen und beraten
- Governance-Richtlinien definieren und überwachen
- Anwendungen betreiben und aktualisieren

**Prozess Outputs an Kunde**

- Betriebsbereite Plattform → Fachbereiche
- Erstellte Fachanwendungen → Anwender
- Schulungsunterlagen → Fachbereiche
- Governance-Report → IT-Leitung
- Betriebsdaten → Application Management

**Prozess Beteiligte / Rollen**

- Platform Manager
- Citizen Developer (Fachbereich)
- IT Security Manager
- Solution Architect

**Unterstützende IT-Systeme**

- Low Code/No Code Plattform (z.B. Power Apps)
- IAM-/Access-Tool
- Dokumentenmanagement

**Kritische Erfolgsfaktoren**

- Klare Governance-Regeln
- Ausreichende Schulung der Fachbereiche
- Sichere Integration in IT-Landschaft
- Akzeptanz bei Fachbereichen
