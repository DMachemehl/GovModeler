# Prozess-Steckbrief IT Problem Management

**Bezug zu IT-Strategie /Auftrag**

- Nachhaltige Stabilisierung der IT-Services im Sinne serviceorientierter Leistungserbringung
- Reduzierung von Störungen durch proaktive Ursachenbeseitigung (Digital MoveS)
- Grundlage für kontinuierliche Verbesserung der IT-Servicequalität

**Prozesszweck** Ursächliche Identifikation, Diagnose und Behebung unbekannter Fehlerursachen zur nachhaltigen Vermeidung wiederkehrender Störungen.

**Prozessziele, KPI, Werte**

- Anzahl gelöster Probleme im Zeitraum
- Reduktionsquote wiederkehrender Incidents
- Durchschnittliche Diagnosedauer
- Anteil proaktiv identifizierter Probleme

**Prozess Inputs durch Lieferanten**

- Incidents / Störungsmuster → Incident Management
- Ereignisse / Trends → Event Management
- Konfigurationsdaten → Configuration Management
- Bekannte Fehler / Workarounds → Knowledge Management
- Risikohinweise → IT Security Management

**Prozessaktivitäten**

- Problemidentifikation durchführen
- Ursachendiagnose erstellen
- Fehlerbehebung entwickeln
- Known Error dokumentieren
- Wirksamkeit überwachen

**Prozess Outputs an Kunde**

- Ursachenanalyse → Incident Management
- Known Error Record → Knowledge Management
- Änderungsanforderung → Change Management
- Präventionsmaßnahmen → Continual Service Improvement
- Statusbericht → IT-Management

**Kritische Erfolgsfaktoren**

- Systematische Root-Cause-Analyse
- Enge Verzahnung mit Incident Mgt
- Aktuelle Knowledge-Datenbasis
- Proaktive Fehlervermeidung

**Prozess Beteiligte / Rollen**

- Problem Manager → Steuerung
- Technical Analyst → Diagnose
- Incident Manager → Übergabe
- Change Manager → Umsetzung

**Unterstützende IT-Systeme**

- ITSM-Tool
- Knowledge Datenbank
- Monitoring-Tools
