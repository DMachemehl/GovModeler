# Prozess-Steckbrief Event Management

**Bezug zu IT-Strategie /Auftrag**

- Sicherstellung stabiler, zuverlässiger IT-Services für die Verwaltung (Digital MoveS)
- Grundlage für frühzeitige Störungserkennung und proaktives Handeln
- Beitrag zur kontinuierlichen Verbesserung der IT-Betriebsstabilität

**Prozesszweck** Kontinuierliche Überwachung von IT-Systemen und -Services zur frühzeitigen Erkennung, Klassifizierung und Reaktion auf relevante Ereignisse.

**Prozessziele, KPI, Werte**

- Anzahl erkannter Events pro Kategorie
- Reaktionszeit auf kritische Events
- Anteil automatisiert erkannter Störungen
- Falsch-Positiv-Quote der Meldungen

**Prozess Inputs durch Lieferanten**

- Monitoring-Daten → Technische Architektur / Betrieb
- Schwellwerte / Regeln → Capacity Management
- Konfigurationsdaten → Configuration Management
- Sicherheitsereignisse → SOC / IT Security Management
- Serviceverfügbarkeit → Verfügbarkeits Mgt

**Prozessaktivitäten**

- Ereignisse erfassen
- Ereignisse klassifizieren
- Ereignisse korrelieren
- Reaktion auslösen
- Ereignisse dokumentieren

**Prozess Outputs an Kunde**

- Incident-Trigger → Incident Management
- Problem-Trigger → Problem Management
- Statusmeldungen → Verfügbarkeits Mgt
- Sicherheitsereignis → SOC
- Reporting → IT-Management

**Kritische Erfolgsfaktoren**

- Vollständige Systemüberwachung
- Sinnvolle Schwellwertdefinition
- Automatisierte Ereigniskorrelation
- Klare Eskalationswege

**Prozess Beteiligte / Rollen**

- Event Manager → Steuerung
- IT Operations → Überwachung
- Incident Manager → Übergabe
- Security Manager → Sicherheitsereignisse

**Unterstützende IT-Systeme**

- Monitoring-Tool
- SIEM
- ITSM-Tool
