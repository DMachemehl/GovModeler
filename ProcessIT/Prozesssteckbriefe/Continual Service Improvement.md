# Prozess-Steckbrief: Continual Service Improvement

**Bezug zu IT-Strategie /Auftrag**

- Kontinuierliche Verbesserung der IT-Servicequalität gemäß Digital MoveS
- Sicherstellung nachhaltiger Weiterentwicklung der IT-Leistungserbringung
- Grundlage für serviceorientierte, effiziente Verwaltungs-IT

**Prozesszweck** Systematische Identifikation, Bewertung und Umsetzung von Verbesserungsmaßnahmen für IT-Services und -Prozesse.

**Prozessziele, KPI, Werte**

- Anzahl umgesetzter Verbesserungsmaßnahmen
- Steigerung der Servicequalität (KPI-Trend)
- Nutzerzufriedenheit im Zeitverlauf
- Reifegrad der IT-Prozesse

**Prozess Inputs durch Lieferanten**

- Servicereports → Service Level Mgt
- Incident-/Problemdaten → Incident/Problem Mgt
- Kundenfeedback → Business Relationship Mgt
- Auditergebnisse → IT Qualitäts Mgt
- Kennzahlen → IT Performance Mgt

**Prozessaktivitäten**

- Verbesserungspotenziale identifizieren
- Maßnahmen bewerten & priorisieren
- Verbesserungsplan erstellen
- Umsetzung koordinieren
- Wirksamkeit überprüfen

**Prozess Outputs an Kunde**

- Verbesserungsplan → IT-Management
- Umsetzungsstatus → Prozessverantwortliche
- Aktualisierte KPIs → IT Performance Mgt
- Lessons Learned → Knowledge Mgt
- Reports → Service Level Mgt

**Kritische Erfolgsfaktoren**

- Etablierte Verbesserungskultur
- Verlässliche Datenbasis
- Management-Unterstützung
- Konsequente Nachverfolgung

**Prozess Beteiligte / Rollen**

- CSI Manager
- Service Owner
- Prozessverantwortliche
- IT-Management

**Unterstützende IT-Systeme**

- ITSM-Tool
- Power BI
- Collaboration Tools
