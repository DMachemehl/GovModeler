# Prozess-Steckbrief: IT-Datenschutz betreiben

**Bezug zu IT-Strategie

- Einhaltung von Datenschutzgesetzen (DSGVO, LDSG)
- Schutz personenbezogener Daten
- Vertrauensschutz & Compliance

**Prozesszweck:**  Sicherstellung, dass personenbezogene Daten gemäß geltender Datenschutzgesetze und -richtlinien geschützt und verarbeitet werden sowie Prävention von Datenschutzverstößen.

**Prozessziele, KPI, Werte:** 

- Datenschutzverstöße minimieren
- Compliance-Konformität
- Schulungsquote Mitarbeitende
- Umsetzungsquote Maßnahmen

**Prozess Inputs durch Lieferanten:** 

- Datenschutzvorgaben → Datenschutzbeauftragte
- IT-Systeme & Prozesse → IT-Betrieb
- Verarbeitungstätigkeiten → Fachbereiche
- Audit-Ergebnisse → IT-Security
- Gesetzesänderungen → Compliance

**Prozessaktivitäten:** 

- Datenschutz-Anforderungen analysieren
- Verarbeitungen dokumentieren
- Risiken bewerten
- Schutzmaßnahmen implementieren
- Compliance überwachen & berichten

**Prozess Outputs an Kunde:** 

- Datenschutzvorgaben → IT-Systeme
- Datenschutzberichte → Datenschutzbeauftragte
- Schulungskonzepte → HR
- Maßnahmenpläne → Fachbereiche
- Compliance-Reports → Verwaltungsleitung

**Prozess Beteiligte / Rollen:** 

- Datenschutzbeauftragte → Verantwortung
- IT-Security Manager → Umsetzung
- Prozessverantwortliche → Einhaltung
- Compliance Officer → Überwachung

**Unterstützende IT-Systeme:**

- Datenschutz-Compliance-Tool
- Dokumentenverwaltung
- Excel

**Kritische Erfolgsfaktoren:** 

- Datenschutz-Bewusstsein Mitarbeitende
- Klare Verantwortlichkeiten
- Regelmäßige Schulungen/Audits
- Zeitnahe Incident-Reaktion
