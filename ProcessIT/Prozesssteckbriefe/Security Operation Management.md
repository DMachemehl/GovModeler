# Prozess-Steckbrief: Security Operation Management (SOC betreiben)

**Bezug zu IT-Strategie /Auftrag**

- Schutz der IT-Systeme und Daten der Verwaltung vor Bedrohungen
- Sicherstellung von Verfügbarkeit, Integrität und Vertraulichkeit
- Umsetzung der Sicherheitsvorgaben gemäß IT-Strategie

**Prozesszweck** Betrieb eines zentralen Überwachungs- und Reaktionszentrums zur frühzeitigen Erkennung und Abwehr von IT-Sicherheitsvorfällen.

**Prozessziele, KPI, Werte**

- Erkennungszeit (Mean Time to Detect)
- Reaktionszeit (Mean Time to Respond)
- Anzahl erkannter/abgewehrter Bedrohungen
- Abdeckungsgrad Monitoring

**Prozess Inputs durch Lieferanten**

- Log-/Telemetriedaten → IT Betrieb
- Bedrohungsinformationen → Threat Intelligence
- Sicherheitsvorgaben → IT Security Mgt
- Incident-Meldungen → Incident Mgt
- Assets → Configuration Mgt

**Prozessaktivitäten**

- Bedrohungen überwachen
- Sicherheitsvorfälle erkennen
- Vorfälle analysieren
- Gegenmaßnahmen einleiten
- Vorfälle dokumentieren

**Prozess Outputs an Kunde**

- Sicherheitsvorfälle → Incident Mgt
- Bedrohungsanalysen → IT Security Mgt
- Reports → IT-Management
- Gegenmaßnahmen → IT Betrieb
- Lessons Learned → Knowledge Mgt

**Kritische Erfolgsfaktoren**

- Durchgängiges 24/7-Monitoring
- Qualifiziertes SOC-Personal
- Aktuelle Bedrohungsdaten
- Klare Eskalationsprozesse

**Prozess Beteiligte / Rollen**

- SOC Analyst
- Security Manager
- Incident Manager
- IT Betrieb

**Unterstützende IT-Systeme**

- SIEM-Tool
- Threat Intelligence Platform
- ITSM-Tool
