# Prozess-Steckbrief IT Reporting

**Bezug zu IT-Strategie /Auftrag**

- Transparenz über IT-Leistungen, Kosten und Projekte gemäß Digital MoveS
- Grundlage für fundierte Steuerungsentscheidungen der IT-Leitung
- Unterstützung von Monitoring & Controlling im Steuerungsprozess

**Prozesszweck** Bereitstellung aller IT-Berichte für unterschiedliche Adressaten zur transparenten Steuerung und Entscheidungsunterstützung.

**Prozessziele, KPI, Werte**

- Termingerechte Berichtserstellung
- Datenqualität/Aktualität der Berichte
- Adressatengerechte Aufbereitung
- Zufriedenheit der Berichtsempfänger

**Prozess Inputs durch Lieferanten**

- Kennzahlen/KPI-Daten → Performance Mgt
- Kostendaten → IT Financial Mgt
- Projektstatus → Portfolio Mgt
- Servicedaten → Service Level Mgt
- Berichtsanforderungen → IT-Leitung/Gemeinderat

**Prozessaktivitäten**

- Berichtsanforderungen erheben
- Daten konsolidieren
- Berichte erstellen
- Qualität prüfen
- Berichte verteilen

**Prozess Outputs an Kunde**

- Managementberichte → IT-Leitung
- Statusberichte → Fachbereiche
- Kennzahlenreports → Performance Mgt
- Gremienberichte → Gemeinderat
- Ad-hoc-Reports → Stakeholder

**Prozess Beteiligte / Rolle**

- Reporting Manager
- Data Analyst
- IT-Leitung
- Fachbereiche

**Unterstützende IT-Systeme**

- QlikSense/Power BI
- Excel
- SAP/ERP

**Kritische Erfolgsfaktoren**

- Verlässliche Datenbasis
- Klare Berichtsstruktur
- Automatisierte Datenerfassung
- Zielgruppengerechte Aufbereitung
